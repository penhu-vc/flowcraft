## 📊 執行摘要

FlowCraft 是一個基於 Vue Flow 的工作流自動化引擎，目前已實作 **40+ 個節點**（包含 registry 中定義和部分實作的節點），涵蓋觸發器、AI 工具、資料處理、媒體操作等多個類別。

**專案特色**：
- ✅ 完整的工作流執行引擎（WorkflowEngine.ts）
- ✅ 支援條件執行、級聯停用、節點快取
- ✅ 觸發器輪詢機制（Fallback/Sequential 模式）
- ✅ WebSocket 實時進度推送
- ✅ 強大的 AI 整合（Gemini、NotebookLM、自訂 AI pipeline）
- ✅ YouTube 生態系統深度整合

**本報告目標**：
1. 分析現有節點的完整性
2. 參考 Zapier、n8n、Make.com 等平台識別功能缺口
3. 提出優先級建議（quick wins vs. long-term goals）
4. 提供具體實作指南

#### Step 2: Executor 實作（`/server/src/executors/http-request.ts`）

```typescript
type EmitFn = (event: string, data: unknown) => void

export interface HttpRequestConfig {
  url: string
  method: 'GET' | 'POST' | 'PUT' | 'DELETE'
  headers?: string
  body?: string
  timeout?: number
}

export async function executeHttpRequest(
  config: Record<string, unknown>,
  emit: EmitFn
): Promise<{ body: any; status: number; headers: any; success: boolean }> {

  // 1. 參數驗證
  const { url, method = 'GET', headers, body, timeout = 30000 } = config as HttpRequestConfig

  if (!url) throw new Error('HTTP Request: url is required')

  emit('node:log', { message: `🌐 HTTP ${method} ${url}` })

  try {
    // 2. 解析 headers
    let parsedHeaders: Record<string, string> = {}
    if (headers) {
      try {
        parsedHeaders = JSON.parse(headers)
      } catch {
        throw new Error('Headers must be valid JSON')
      }
    }

    // 3. 解析 body
    let parsedBody: any = undefined
    if (body && (method === 'POST' || method === 'PUT')) {
      try {
        parsedBody = JSON.parse(body)
      } catch {
        throw new Error('Body must be valid JSON')
      }
    }

    // 4. 發送請求（with timeout）
    const controller = new AbortController()
    const timeoutId = setTimeout(() => controller.abort(), timeout)

    const response = await fetch(url, {
      method,
      headers: parsedHeaders,
      body: parsedBody ? JSON.stringify(parsedBody) : undefined,
      signal: controller.signal
    })

    clearTimeout(timeoutId)

    // 5. 解析回應
    const responseHeaders: Record<string, string> = {}
    response.headers.forEach((value, key) => {
      responseHeaders[key] = value
    })

    let responseBody: any
    const contentType = response.headers.get('content-type')

    if (contentType?.includes('application/json')) {
      responseBody = await response.json()
    } else {
      responseBody = await response.text()
    }

    const success = response.ok

    emit('node:log', {
      message: `${success ? '✅' : '❌'} ${response.status} ${response.statusText}`
    })

    return {
      body: responseBody,
      status: response.status,
      headers: responseHeaders,
      success
    }

  } catch (error) {
    const message = error instanceof Error ? error.message : String(error)

    if (message.includes('aborted')) {
      throw new Error(`HTTP Request: timeout after ${timeout}ms`)
    }

    emit('node:log', { message: `❌ Request failed: ${message}` })
    throw error
  }
}
```

#### Step 3: 註冊到 Executor Switch（`/server/src/executor.ts`）

```typescript

export async function executeNode(
  nodeType: string,
  config: Record<string, unknown>,
  emit: EmitFn

  switch (nodeType) {
    // ... 其他 cases
```

### 5.4 Logic & Flow Control（邏輯控制）

#### A. Filter（⭐ 最高優先級）

**用途**：條件過濾資料

**實作方式**：
- 支援運算子：`==`, `!=`, `>`, `<`, `>=`, `<=`, `contains`, `startsWith`, `endsWith`, `regex`
- 布林輸出：`passed` / `rejected`

**範例場景**：
- 過濾符合條件的資料
- 分流處理（價格 > 1000 走高價路線）
- 錯誤處理分支

**技術細節**：
```typescript
const { field, operator, value } = config
const fieldValue = inputs[field]

let passed = false

switch (operator) {
  case 'equals':
    passed = fieldValue === value
    break
  case 'contains':
    passed = String(fieldValue).includes(value)
    break
  case 'gt':
    passed = Number(fieldValue) > Number(value)
    break
  // ... 其他運算子
}

return {
  passed: passed ? inputs : null,
  rejected: !passed ? inputs : null
}
```

**注意**：目前 WorkflowEngine 已支援空值跳過執行，但缺少明確的 Filter 節點讓用戶配置條件。

#### B. Switch（⭐ 高優先級）

**用途**：多條件分支路由

**實作方式**：
- 根據欄位值路由到不同分支
- 支援 default 分支

**範例場景**：
- 根據訂單狀態分流處理
- 根據錯誤類型執行不同邏輯
- 多語言處理分支

**技術細節**：
```typescript
const { field, cases } = config
const fieldValue = inputs[field]

// 找出匹配的 case
const matchedCase = cases.find(c => c.value === fieldValue)

if (matchedCase) {
  return { [matchedCase.output]: inputs }
} else {
  return { default: inputs }
}
```

**Outputs 設計**：
```typescript
outputs: [
  { key: 'case_1', label: 'Case 1', type: 'any' },
  { key: 'case_2', label: 'Case 2', type: 'any' },
  { key: 'case_3', label: 'Case 3', type: 'any' },
  { key: 'default', label: 'Default', type: 'any' }
]
```

#### C. Delay（⭐ 高優先級）

**用途**：暫停工作流 N 秒

**實作方式**：
- 使用 `setTimeout` + `Promise`
- 支援秒/分鐘

**範例場景**：
- API 速率限制（每秒最多 1 次請求）
- 等待外部處理完成
- 模擬用戶行為

**技術細節**：
```typescript
const { seconds } = config

emit('node:log', { message: `⏳ 等待 ${seconds} 秒...` })

await new Promise(resolve => setTimeout(resolve, seconds * 1000))

emit('node:log', { message: '✅ 等待結束' })

return { output: inputs }
```

#### D. Code (JavaScript)（⭐ 高優先級，需沙箱）

**用途**：執行自訂 JavaScript 程式碼

**實作方式**：
- 使用 `vm2` 或 `isolated-vm` 沙箱
- 限制執行時間
- 限制記憶體

**範例場景**：
- 複雜的資料轉換
- 自訂驗證邏輯
- 臨時的數學運算

**技術細節**：
```typescript
import { VM } from 'vm2'

const vm = new VM({
  timeout: 5000,              // 5 秒 timeout
  sandbox: {
    input: inputs,            // 上游資料
    console: {
      log: (msg: any) => emit('node:log', { message: String(msg) })
    }
  }
})

const code = `
  // 用戶程式碼
  ${config.code}

  // 必須返回結果
  return result
`

const result = vm.run(code)

return { result }
```

**安全注意事項**：
- 禁用 `require()` / `import`
- 限制檔案系統存取
- 限制網路存取
- 限制執行時間和記憶體
