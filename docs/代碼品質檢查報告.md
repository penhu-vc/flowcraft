## 2. 代碼品質分析

### 2.1 TypeScript 使用

**優點**:
- ✅ 大部分檔案使用 TypeScript，型別定義基本完整
- ✅ Interface 定義清晰（如 `Workflow`, `WorkflowNode`, `EmitFn`）
- ✅ 自定義型別使用良好（如 `ExecutionStatus`, `NodeStatus`）

**問題**:
- 🟡 **中風險**: `any` 使用過多（67 處）
  - **位置**:
    - `server/src/index.ts:180` - `workflows.find((w: any) => ...)`
    - `server/src/executor.ts:60` - `let items: any[] = []`
    - `src/composables/useWorkflowExecution.ts:27-124` - Socket 事件回調全部使用 `any`
  - **影響**: 失去型別安全性，容易產生運行時錯誤
  - **建議**: 定義明確的 interface
    ```typescript
    // ❌ 錯誤
    socket.on('node:done', (data: any) => { ... })

    // ✅ 正確
    interface NodeDoneEvent {
      nodeId: string
      nodeType: string
      result: unknown
      duration?: number
      cached?: boolean
    }
    socket.on('node:done', (data: NodeDoneEvent) => { ... })
    ```

- 🟢 **低風險**: `Record<string, any>` 使用（主要在配置物件）
  - **位置**: `server/src/index.ts:92`, `server/src/execution/WorkflowEngine.ts`
  - **說明**: 配置物件使用 `Record<string, any>` 可以接受，但建議改為 `Record<string, unknown>`

## 4. 安全性問題（🔴 高風險優先）

### 🔴 高風險

#### 4.1 硬編碼敏感 Token
**位置**: `server/src/index.ts:228`
```typescript
const botToken = process.env.TELEGRAM_BOT_TOKEN || '8299044370:AAGSx2Hkgf_VOTIfB9tNAlQ8W1RddFrStRw'
```
- **風險**: Telegram Bot Token 直接寫在代碼中，若推送到公開 Git 會洩漏
- **影響**: 攻擊者可接管 Bot，發送垃圾訊息
- **建議**:
  1. 移除硬編碼值，改為必須從環境變數讀取
  2. 立即 revoke 該 Token 並重新生成
  3. 檢查 Git history，確保未洩漏到公開倉庫
  ```typescript
  // ✅ 正確做法
  const botToken = process.env.TELEGRAM_BOT_TOKEN
  if (!botToken) {
    throw new Error('TELEGRAM_BOT_TOKEN is required')
  }
  ```

#### 4.2 GCP 憑證儲存在檔案系統
**位置**: `server/src/index.ts:245`, `server/data/gcp-credentials.json`
- **風險**: 敏感憑證儲存在專案目錄，容易誤上傳
- **影響**: GCP 專案被接管，產生高額費用
- **建議**:
  1. 確保 `server/data/` 在 `.gitignore` 中
  2. 使用環境變數或秘密管理服務
  3. 新增 `.gitignore` 檢查腳本

#### 4.3 無輸入驗證和注入防護
**位置**: `server/src/index.ts:179`, `server/src/telegram-webhook.ts`
- **風險**:
  - JSON.parse 未驗證輸入，可能導致 DoS
  - Telegram callback_data 未驗證，可能注入惡意代碼
- **建議**: 使用驗證庫（如 `zod`, `joi`）

### 🟡 中風險

#### 4.4 CORS 配置過於寬鬆
**位置**: `server/src/index.ts:15-20`
- **風險**: 允許多個本地 origin，開發環境可接受，但生產環境需要限制
- **建議**: 根據環境變數動態設定

#### 4.5 缺少速率限制
**位置**: `server/src/index.ts` - 所有 API 端點
- **風險**: 可能遭受 DoS 攻擊或 API 濫用
- **建議**: 使用 `express-rate-limit` 中間件

### 🟢 低風險

#### 4.6 環境變數未加密
**位置**: `.env.local` (若存在)
- **建議**: 使用 `dotenv-vault` 或類似工具加密敏感變數
